Como leer estas fichas

Etiqueta completa

Veras la etiqueta escrita como una frase corta, por ejemplo: <ul> ... </ul>.

Idea simple

Debajo te digo que hace y con que otras etiquetas suele ir.

Sin bloques largos

Saque la parte de codigo largo para que todo quede mas rapido de mirar.

Mas variedad

Ahora hay muchas mas etiquetas separadas por tema para que tengas una referencia mas completa.

Idea rapida para recordar HTML

Si HTML es el esqueleto de la pagina, sus etiquetas son las piezas que ordenan cada parte. Con muy pocas etiquetas ya puedes armar titulos, textos, listas, imagenes, tablas y formularios.

Si quieres ver la estructura completa explicada de forma simple, recorre estas fichas: estan separadas por base, estructura, texto, listas, multimedia, tablas, formularios y extras.

Base del documento

<!doctype html>

Le avisa al navegador que el archivo usa HTML moderno.

Va siempre arriba de todo.

<html> Todo el sitio vive aqui. </html>

Es la caja mas grande de la pagina.

Adentro suelen ir head y body.

<head> Configuracion del archivo. </head>

Guarda datos que ayudan a la pagina, pero casi nunca se ven.

Suele contener meta, title y link.

<meta charset="UTF-8">

Ayuda a que letras y simbolos se vean bien.

No suele llevar cierre.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Hace que la pagina se adapte mejor al celular.

Tambien va dentro de head.

<title>Nombre de la pestana</title>

Es el titulo que aparece arriba en la pestana del navegador.

Va dentro de head.

<link rel="stylesheet" href="estilos.css">

Conecta tu HTML con un archivo CSS.

Suele ir dentro de head y no necesita cierre.

<style> Reglas CSS cortas. </style>

Sirve para escribir CSS directamente dentro del HTML.

Se usa poco en proyectos ordenados, porque normalmente el CSS va en otro archivo.

<script> Codigo de interaccion. </script>

Se usa para agregar acciones o comportamiento a la pagina.

Suele ir al final de body o dentro de head.

<base href="https://ejemplo.com/">

Le da una ruta base a todos los enlaces del documento.

Se usa poco, pero vive dentro de head.

<body> Todo lo que si se ve. </body>

Guarda el contenido visible de la pagina.

Aqui viven titulos, textos, imagenes, listas y formularios.

<noscript> Mensaje si no hay JavaScript. </noscript>

Muestra un aviso cuando cierta parte interactiva no funciona.

Se usa a veces para dar una aclaracion extra.

Estructura y contenedores

<header> Logo, titulo o menu. </header>

Es la parte de arriba de una pagina o de una seccion.

Suele contener h1, nav o un logo.

<nav> Enlaces importantes. </nav>

Agrupa links de navegacion.

Suele llevar varios a adentro.

<main> Contenido principal. </main>

Guarda lo mas importante de la pagina.

Suele envolver section, article y div.

<section> Un tema de la pagina. </section>

Sirve para separar grandes bloques por tema.

Dentro pueden ir titulos, textos, listas o tarjetas.

<article> Una pieza con sentido propio. </article>

Se usa para noticias, posts o tarjetas que podrian vivir solas.

Suele ir dentro de una section.

<aside> Informacion secundaria. </aside>

Sirve para tips, avisos o contenido lateral.

Suele acompanar a un bloque principal.

<footer> Cierre del bloque o del sitio. </footer>

Es la parte final donde suelen ir datos extra o contacto.

Puede estar al final de la pagina o de una tarjeta.

<div> Suele envolver sections, listas o tarjetas. </div>

Es una caja general para agrupar contenido.

Se usa cuando necesitas ordenar cosas sin darles un significado especial.

<span> Una parte pequena del texto. </span>

Sirve para marcar una palabra o pedazo corto dentro de una frase.

Suele ir dentro de p, h1 o button.

<address> Datos de contacto. </address>

Sirve para mostrar direccion, mail, telefono o informacion de contacto.

Suele ir en un footer o en una seccion de contacto.

Texto y titulos

<h1> Titulo principal. </h1>

Es el titulo mas importante de la pagina.

Despues vienen h2, h3 y asi hasta h6.

<h2> Subtitulo del tema. </h2>

Se usa para dividir el contenido en partes grandes.

Suele ir dentro de una section.

<p> Texto comun de explicacion. </p>

Guarda un parrafo normal.

Suele ir debajo de titulos o dentro de tarjetas.

<strong> Esto es muy importante. </strong>

Resalta una parte importante del texto.

Suele ir dentro de un p.

<em> Esto merece enfasis. </em>

Da enfasis a una palabra o parte de la frase.

Tambien suele ir dentro de un p.

<small> Aclaracion corta. </small>

Sirve para notas pequenas o texto secundario.

Suele acompanar precios, fechas o advertencias.

<mark> Texto resaltado. </mark>

Pinta una parte del texto como si la subrayaras con color.

Va muy bien dentro de un p.

<abbr title="Explicacion de la sigla">Sigla</abbr>

Sirve para siglas o abreviaturas.

El titulo suele explicar el significado completo.

<time> 20 de abril </time>

Guarda una fecha u hora.

Suele ir dentro de un p o una tarjeta.

<blockquote> Una cita larga. </blockquote>

Se usa para frases destacadas o testimonios.

Suele ir acompanada por cite.

<cite> Fuente de la cita. </cite>

Nombra la obra, persona o lugar de donde sale una cita.

Suele ir cerca de un blockquote.

<br>

Hace un salto de linea corto.

No suele llevar cierre.

<hr>

Marca una separacion entre temas.

Se usa para cortar visualmente un bloque del siguiente.

<sup> Texto arriba. </sup>

Sube un texto pequeno.

Sirve en potencias o notas.

<sub> Texto abajo. </sub>

Baja un texto pequeno.

Sirve en formulas como H2O.

<b> Texto destacado visualmente. </b>

Hace que una parte del texto se vea mas fuerte.

Se parece a strong, pero suele usarse mas por apariencia que por importancia.

<i> Texto con tono distinto. </i>

Sirve para palabras extranjeras, nombres especiales o tonos diferentes.

Se parece a em, pero no siempre significa enfasis.

<u> Texto subrayado. </u>

Subraya una parte del texto.

Conviene usarlo con cuidado para no confundirlo con un enlace.

<del> Texto eliminado. </del>

Muestra algo que ya no vale o fue quitado.

Va bien para precios viejos o cambios en un texto.

<ins> Texto agregado. </ins>

Marca una parte nueva que fue agregada.

A veces se usa junto con del.

<q> Una cita corta. </q>

Sirve para citas breves dentro de una frase.

Se usa cuando no hace falta un blockquote completo.

<s> Texto que ya no aplica. </s>

Tacha una parte del texto.

Es muy comun en ofertas o contenido viejo.

<bdi> Texto aislado. </bdi>

Ayuda cuando mezclas textos que pueden ir en distinto sentido.

Se usa poco, pero sirve para nombres o palabras en otros idiomas.

<bdo dir="rtl"> Texto invertido. </bdo>

Fuerza la direccion del texto.

Se usa sobre todo en idiomas que no se escriben de izquierda a derecha.

Listas y enlaces

<a> Ir a otra pagina. </a>

Crea un enlace.

Puede llevar a otra pagina, a una seccion o a un archivo.

<ul> Suele contener varios li. </ul>

Es una lista desordenada, o sea, con vinetas.

Suele ir dentro de un div o una section.

<ol> Suele contener varios li numerados. </ol>

Es una lista ordenada.

Va bien para pasos o instrucciones.

<li> Un punto de la lista. </li>

Es cada elemento individual de una lista.

Suele ir dentro de ul o ol.

<dl> Terminos con su explicacion. </dl>

Es una lista especial para definiciones.

Suele contener dt y dd.

<dt> Palabra a explicar. </dt>

Es el termino principal dentro de una lista de definiciones.

Suele ir antes de dd.

<dd> Explicacion de esa palabra. </dd>

Da el significado o la descripcion del termino.

Suele ir debajo de un dt.

<menu> Grupo de opciones. </menu>

Es otra forma de agrupar opciones o acciones.

Se usa menos que ul, pero existe.

Imagen, audio y video

<img src="foto.jpg" alt="descripcion">

Muestra una imagen.

Suele ir dentro de div, figure o tarjetas.

<figure> Imagen con contexto. </figure>

Junta una imagen o grafico con su texto de apoyo.

Suele contener img y figcaption.

<figcaption> Explicacion de la imagen. </figcaption>

Describe lo que se ve en la imagen o video.

Suele ir dentro de figure.

<picture> Varias versiones de una imagen. </picture>

Sirve cuando quieres cargar una imagen distinta segun la pantalla.

Suele contener source e img.

<source src="archivo.mp4" type="video/mp4">

Le dice al navegador que archivo debe usar.

Suele ir dentro de audio, video o picture.

<audio> Sonido para reproducir. </audio>

Sirve para musica, voz o efectos de audio.

Suele contener una o varias etiquetas source.

<video> Video en la pagina. </video>

Muestra un video sin salir del sitio.

Tambien suele contener source.

<iframe> Otra pagina adentro. </iframe>

Inserta un mapa, un formulario o un contenido externo.

Se usa mucho para videos y mapas.

<canvas> Zona para dibujar. </canvas>

Reserva un espacio donde luego se puede dibujar con codigo.

Se usa a veces en juegos, graficos o efectos.

<track> Subtitulos del video. </track>

Agrega subtitulos o pistas de texto a un video.

Suele ir dentro de video.

<svg> Dibujo vectorial. </svg>

Sirve para iconos o dibujos que no pierden calidad al agrandarse.

Se usa mucho en logos, iconos y graficos simples.

Tablas

<table> Datos en filas y columnas. </table>

Es la caja principal de una tabla.

Suele contener caption, thead, tbody y filas.

<caption> Titulo de la tabla. </caption>

Explica de que trata la tabla.

Suele ir arriba dentro de table.

<thead> Nombres de columnas. </thead>

Guarda la parte superior de la tabla.

Suele contener filas con th.

<tbody> Datos principales. </tbody>

Guarda el cuerpo de la tabla.

Suele contener varias filas tr.

<tfoot> Cierre o resumen. </tfoot>

Se usa para totales o notas finales.

Va a veces, no siempre.

<tr> Una fila de la tabla. </tr>

Crea una fila horizontal.

Dentro suelen ir th o td.

<th> Titulo de columna o fila. </th>

Marca una celda de encabezado.

Suele ir dentro de thead o al inicio de una fila.

<td> Dato de la tabla. </td>

Es una celda comun.

Va dentro de una fila tr.

<colgroup> Grupo de columnas. </colgroup>

Sirve para tocar o identificar columnas enteras de una tabla.

Suele contener una o varias etiquetas col.

<col>

Representa una columna de la tabla.

Se usa dentro de colgroup y no suele llevar cierre.

Formularios

<form> Varios campos para completar. </form>

Es la caja principal del formulario.

Suele contener label, input, select y button.

<fieldset> Grupo de campos relacionados. </fieldset>

Sirve para juntar varios campos del mismo tema.

Suele contener legend y varios controles.

<legend> Titulo del grupo. </legend>

Nombra un bloque de campos.

Suele ir dentro de fieldset.

<label> Nombre del campo. </label>

Le explica a la persona que debe escribir o elegir.

Suele ir justo antes de un input, select o textarea.

<input type="text">

Es una caja corta para escribir datos.

Puede ser texto, email, password, fecha y mucho mas.

<textarea> Mensaje largo. </textarea>

Sirve para escribir varias lineas.

Va bien para comentarios o descripciones.

<select> Opciones para elegir. </select>

Crea una lista desplegable.

Suele contener varias etiquetas option.

<option> Una opcion del menu. </option>

Es cada item dentro de un selector.

Suele ir dentro de select.

<button> Enviar o guardar. </button>

Es el boton que activa una accion.

Puede enviar el formulario o hacer otra tarea.

<datalist> Sugerencias para escribir. </datalist>

Le da opciones sugeridas a un campo de texto.

Suele ir conectada a un input.

<optgroup> Grupo de opciones. </optgroup>

Sirve para ordenar un select en grupos.

Va dentro de select y contiene option.

<output> Resultado mostrado. </output>

Muestra un resultado calculado o actualizado.

Se usa poco, pero sirve para formularios interactivos.

Etiquetas utiles que aparecen a veces

<details> Caja que se abre y se cierra. </details>

Sirve para mostrar contenido oculto al principio.

Suele contener summary.

<summary> Titulo del desplegable. </summary>

Es la parte que tocas para abrir un details.

Va dentro de details.

<pre> Texto con espacios respetados. </pre>

Muestra texto tal como fue escrito.

Se usa mucho para codigo o bloques tecnicos.

<code> Parte tecnica o codigo. </code>

Marca un fragmento de codigo o una etiqueta.

Suele ir dentro de un pre o de un p.

<kbd> Tecla del teclado. </kbd>

Sirve para mostrar combinaciones como Ctrl + S.

Suele ir dentro de un p.

<progress> Cuanto falta. </progress>

Muestra el avance de algo.

Va bien para cargas o pasos completados.

<meter> Nivel de algo. </meter>

Mide una cantidad, como bateria o capacidad.

No es para progreso: es para nivel actual.

<dialog> Ventana emergente. </dialog>

Sirve para una caja que aparece encima del contenido.

Se usa menos, pero existe para mensajes o confirmaciones.

<samp> Texto de salida del sistema. </samp>

Muestra una respuesta o mensaje como si viniera de la computadora.

Va bien en ejemplos tecnicos.

<var> Nombre de un valor. </var>

Sirve para mostrar una variable o un dato que puede cambiar.

Se usa en ejemplos tecnicos o matematicos.

<data value="123"> Producto </data>

Guarda un dato visible y otro mas preciso por detras.

Sirve cuando quieres mostrar un texto y tambien guardar su valor exacto.

Propiedades CSS esenciales

Mantengo esta parte como resumen rapido. Si quieres verla comentada linea por linea, puedes abrir el archivo CSS aparte.

padding

Agrega aire dentro de la caja.

margin

Separa la caja de otras por fuera.

border

Dibuja el borde del elemento.

border-radius

Redondea esquinas.

background

Pone un fondo completo, incluso degradado.

background-color

Pinta el fondo con un color liso.

color

Cambia el color del texto.

font-family

Elige la tipografia.

font-size

Cambia el tamano de la letra.

line-height

Da mas o menos aire entre lineas.

text-align

Alinea el texto.

width

Marca el ancho del elemento.

max-width

Pone un limite de ancho.

height

Marca el alto del elemento.

min-width

Pone un ancho minimo.

min-height

Pone un alto minimo.

display

Decide como se comporta la caja.

flex-direction

Dice si los elementos van en fila o columna dentro de flex.

flex-wrap

Permite que los elementos bajen a otra linea.

justify-content

Reparte elementos en el eje principal.

align-items

Alinea elementos en el eje cruzado.

gap

Separa elementos dentro de flex o grid.

grid-template-columns

Define cuantas columnas tendra una grilla.

grid-template-rows

Define las filas de una grilla.

box-shadow

Agrega una sombra.

border-color

Cambia el color del borde.

border-width

Cambia el grosor del borde.

border-style

Decide si el borde es solido, punteado u otro estilo.

position

Da mas control sobre la ubicacion.

top

Mueve una caja desde arriba cuando usa posicion especial.

right

Ubica una caja tomando como referencia el lado derecho.

left

Mueve una caja hacia la izquierda o la ubica desde ese lado.

bottom

Ubica una caja tomando como referencia la parte de abajo.

z-index

Decide que elemento queda por encima de otro.

overflow

Controla que pasa si el contenido se sale.

opacity

Hace un elemento mas transparente o mas solido.

object-fit

Ajusta como se acomoda una imagen o video dentro de su caja.

transition

Hace suaves los cambios de estilo.

transform

Mueve, gira o agranda un elemento.

cursor

Cambia el icono del mouse.

font-weight

Hace la letra mas fina o mas gruesa.

text-transform

Pasa el texto a mayusculas, minusculas u otro formato.

letter-spacing

Separa mas o menos las letras.

text-decoration

Agrega o quita lineas como el subrayado.

background-image

Pone una imagen de fondo.

background-size

Decide como se acomoda el fondo dentro de la caja.

background-position

Mueve la imagen de fondo dentro de la caja.

border-collapse

Une los bordes de una tabla para que no se vean dobles.

white-space

Controla si el texto respeta espacios y saltos.

outline

Dibuja una linea exterior, muy usada en focos.
