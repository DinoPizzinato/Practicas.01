Fundamentos Selectores Propiedades HTML

Fundamentos primero

Primero entiende como trabajan HTML y CSS juntos

Abrir bloque completo

Antes de memorizar propiedades o copiar ejemplos, conviene entender que HTML define la estructura del documento y CSS decide como se presenta esa estructura en pantalla. Cuando esa base se entiende bien, el resto deja de sentirse como una lista de trucos sueltos.

HTML: estructura y significado

HTML no se usa para "hacerlo ver lindo", sino para decirle al navegador que una pieza es un titulo, un parrafo, una lista, un formulario, un enlace o una seccion. Esa estructura importa porque CSS no estiliza pixeles sueltos: estiliza elementos HTML concretos y las relaciones que tienen entre si.

Si el HTML esta bien armado, luego es mas facil seleccionar, ordenar y mantener el contenido. Si el HTML esta confuso, CSS termina lleno de parches.

CSS: presentacion, reglas y layout

CSS toma esa estructura y decide color, espaciado, tamanos, bordes, posicion, distribucion y respuesta a distintos anchos de pantalla. Pero no aplica todo al azar: primero tiene que decidir que regla coincide con cada elemento y cual gana cuando varias compiten.

Por eso estudiar CSS no es solo aprender propiedades. Tambien es entender cascada, especificidad, herencia, box model y flujo normal.

Que hace el navegador con todo eso

El navegador lee el HTML, construye una estructura interna del documento, lee el CSS y prueba que reglas coinciden con cada elemento. Despues calcula estilos finales, tamanos, posiciones y recien ahi dibuja la pagina.

Cuando algo falla, casi siempre el problema esta en alguna parte de esa cadena: el HTML, el selector, la regla que gano, la caja o el contexto donde se acomoda el elemento.

Bloque aparte

Fundamentos y conceptos de CSS para estudiar

Despues seguir con propiedades

Aprender propiedades ayuda, pero entender como piensa CSS es lo que realmente te deja resolver problemas sin depender de memoria pura.

Esta seccion esta pensada para estudiar: conceptos base, ejemplos cortos y preguntas utiles para entender por que una regla funciona.

Antes de bajar a propiedades, esta es la idea completa

HTML pone la estructura y el significado del contenido. CSS entra despues para leer esa estructura y decidir como se ve cada pieza. Eso significa que, si no entiendes que elemento estas tocando en HTML, tambien sera dificil entender por que CSS no responde como esperas.

El navegador no piensa en "hacerlo lindo" o "hacerlo feo". Piensa en reglas. Primero verifica que selector coincide con que elemento, luego resuelve cascada y especificidad, calcula herencia, mide la caja, acomoda todo en el flujo normal o en un layout especial y al final pinta la interfaz. Esa secuencia es la base real para estudiar CSS y HTML con sentido.

1. Cascada

Si dos reglas compiten, CSS decide cual gana

La cascada no significa solo "gana la ultima". Primero importa si ambas reglas llegan al mismo elemento y si tienen un peso parecido. Si empatan, gana la que aparece mas abajo.

Cuando el navegador encuentra varias reglas para el mismo elemento, no las aplica todas con el mismo valor final. La cascada es justamente el sistema que decide que declaracion termina viva. Entender esto evita tocar una regla esperando un cambio cuando, en realidad, otra mas fuerte o mas abajo la esta pisando.

Idea

mismo selector
misma propiedad
gana la ultima regla

CSS

.titulo {
  color: steelblue;
}

.titulo {
  color: tomato;
}

Pregunta para estudiar: las dos reglas apuntan al mismo elemento? Si no, la cascada ni siquiera entra en juego.

2. Especificidad

Un selector mas especifico puede ganarle a otro

En general, un #id pesa mas que una .clase y una clase pesa mas que una etiqueta. Entender esto evita pelearte con CSS a base de reglas cada vez mas largas.

La especificidad no mide que selector esta "mejor escrito", sino cuanta precision usa para apuntar. Si te acostumbras a resolver todo con selectores demasiado pesados, despues cada cambio necesita todavia mas peso y el CSS se vuelve dificil de mantener.

Orden mental

#id  >  .clase  >  etiqueta

CSS

h1 {
  color: steelblue;
}

.hero h1 {
  color: seagreen;
}

#inicio h1 {
  color: tomato;
}

Regla util: intenta ganar por claridad y estructura, no por fuerza bruta.

3. Herencia

Algunas propiedades bajan a los hijos y otras no

Colores, tipografias y line-height suelen heredarse. En cambio, medidas, bordes, margenes o fondos no se copian automaticamente.

La herencia explica por que muchas veces un texto ya toma una tipografia o un color sin que escribas la regla en cada hijo. Pero tambien explica por que un borde, un fondo o un margen no aparecen magicamente en todos los descendientes. Saber que viaja y que no viaja te ordena mucho la cabeza.

Idea

color y font-family suelen heredarse
border y margin no

CSS

body {
  color: #243447;
  font-family: "Trebuchet MS", sans-serif;
}

.card {
  border: 1px solid #d8d1c4;
}

Cuando un texto no toma el color esperado, revisa si realmente hereda o si otra regla lo esta sobrescribiendo.

4. Box model

Toda caja tiene contenido, padding, borde y margen

El modelo de caja explica por que un elemento ocupa cierto espacio. Muchas veces el problema no es el ancho, sino el padding, el border o el margin que se le suman.

Cuando dices que una caja mide cierto ancho, eso no siempre significa que ese sera el espacio visual final. Depende de si el padding y el borde quedan sumados por fuera o incluidos dentro de la medida. Por eso muchisimos problemas de layout no son de "ancho roto", sino de caja mal entendida.

Capas

contenido
padding
border
margin

CSS

* {
  box-sizing: border-box;
}

.card {
  width: 320px;
  padding: 24px;
  border: 2px solid #1f5c4a;
  margin: 16px auto;
}

box-sizing: border-box; suele simplificar mucho el calculo visual de las cajas.

5. Flujo y display

Primero existe un flujo normal, despues lo modificas

CSS no arranca con flex ni grid. Primero hay un flujo por defecto donde algunos elementos ocupan toda la linea y otros viven dentro del texto.

El flujo normal es la manera basica en que el navegador acomoda el documento antes de que tu fuerces otro comportamiento. Si no entiendes como funciona un elemento como block, inline o inline-block, despues cuesta entender por que no acepta ancho, por que baja de linea o por que se mezcla con el texto.

Idea

block: baja de linea
inline: sigue dentro del texto
inline-block: mezcla ambos

CSS

p {
  display: block;
}

span {
  display: inline;
}

.badge {
  display: inline-block;
}

Mucha confusion con anchos o alturas nace de olvidar que un elemento sigue siendo inline o block.

6. Flex y Grid

Flex ordena un eje; Grid piensa filas y columnas a la vez

Flexbox va genial para grupos en fila o columna. Grid brilla cuando quieres controlar una composicion completa con varias columnas y filas.

Flexbox suele servir cuando la pregunta es como alinear, repartir o separar elementos en un solo eje. Grid se vuelve mas claro cuando ya estas pensando en una composicion completa, con filas y columnas al mismo tiempo. Elegir bien entre ambos te simplifica mucho el codigo.

Regla rapida

Flex: una dimension
Grid: dos dimensiones

CSS

.menu {
  display: flex;
  gap: 12px;
}

.galeria {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}

Si te pierdes, pregunta simple: estoy acomodando items en un eje o diseniando una cuadricula completa?

7. Responsive

Un buen CSS se adapta al espacio disponible

Responsive no es solo media query. Tambien entran en juego unidades relativas, anchos fluidos y limites minimos o maximos.

La web no vive en una sola pantalla fija. El mismo HTML puede verse en un celular angosto, una tablet o un monitor grande. El CSS responsive busca que el contenido responda al espacio disponible sin romper lectura, jerarquia ni comodidad de uso.

Pistas utiles

rem, %, vw, min(), max(), clamp()

CSS

.hero {
  padding: clamp(16px, 4vw, 48px);
}

.card {
  width: min(100%, 420px);
}

@media (max-width: 768px) {
  .layout {
    grid-template-columns: 1fr;
  }
}

Estudiar responsive es aprender a pensar en rangos, no en una sola pantalla fija.

8. Posicion y capas

Posicionar no es mover por capricho, es cambiar la referencia

relative prepara una caja. absolute ubica un hijo respecto a un ancestro posicionado. fixed se pega a la ventana y sticky mezcla flujo normal con fijacion al scroll.

Cambiar position no es solo "mover cosas". Tambien cambia desde donde el navegador calcula la referencia de esa caja y si sigue participando o no del flujo normal. Por eso un elemento absoluto puede verse correcto visualmente, pero alterar por completo la logica del layout si se usa sin entenderlo.

Idea

relative = referencia
absolute = sale del flujo
sticky = se pega al hacer scroll

CSS

.panel {
  position: relative;
}

.badge {
  position: absolute;
  top: 12px;
  right: 12px;
  z-index: 2;
}

.barra {
  position: sticky;
  top: 0;
}

Si algo queda debajo de otra capa, revisa contexto, orden y z-index, no solo el valor numerico.

9. Estados

CSS tambien reacciona a estados y contexto

Las pseudoclases permiten cambiar el estilo cuando el usuario interactua o cuando un elemento ocupa cierta posicion dentro de sus hermanos.

CSS no solo pinta cosas quietas. Tambien reacciona a estados del mouse, del teclado, del formulario y del lugar que ocupa un elemento dentro de la estructura. Eso permite interfaces mas ricas sin tener que usar JavaScript para cada detalle visual.

Casos comunes

:hover
:focus-visible
:active
:first-child

CSS

.boton:hover {
  transform: translateY(-2px);
}

.campo:focus-visible {
  outline: 2px solid #1f5c4a;
}

.lista li:first-child {
  font-weight: bold;
}

Si quieres que algo sea usable con teclado, no te quedes solo con :hover.

10. Debug

Cuando algo no funciona, casi siempre falla una idea base

La mayoria de los bugs de CSS no se resuelven memorizando mas propiedades, sino revisando selector, flujo, caja, contexto y estados.

Depurar CSS es reconstruir la cadena de decisiones del navegador: que elemento seleccionaste, que regla gano, como se calculo la caja y dentro de que contexto se dibujo. Cuando entiendes esa cadena, dejas de probar propiedades al azar y empiezas a corregir con criterio.

Chequeo rapido

1. Apunta al elemento correcto?
2. Otra regla pesa mas?
3. Falta ancho, alto o display?
4. El padre recorta con overflow?
5. Depende de hover, focus o media query?

CSS util

.debug * {
  outline: 1px dashed rgba(31, 92, 74, 0.2);
}

.debug img {
  background-color: rgba(31, 92, 74, 0.08);
}

Buen habito de estudio: antes de tocar diez reglas, intenta nombrar que concepto base esta fallando.

Propiedades CSS esenciales

Nivel

Categoria

Aqui cada propiedad viene con un ejemplo completo. Cuando una regla depende de otra, tambien te la dejo escrita para que el ejemplo funcione tal como esta.

Para empezar con CSS, lo mas importante es entender estas tres formas de llamar cosas: por etiqueta, por clase y por id. Desde ahi ya puedes pasar a ideas mas propias de CSS como estados, variables, reglas responsive y atajos de escritura.

1. Etiqueta directa

Cuando quieres tocar una etiqueta tal cual aparece

Si en HTML ves body, p, h1 o form, en CSS la llamas escribiendo ese mismo nombre, sin punto y sin numeral.

HTML

<body>...</body>

CSS

body {
  font-family: "Trebuchet MS", sans-serif;
}

Regla rapida: si es el nombre de una etiqueta, la copias tal cual.

2. Clase

Cuando quieres repetir el mismo estilo en varias cajas

Las clases llevan class="..." en HTML. En CSS se llaman con un punto adelante.

HTML

<div class="filtros">...</div>

CSS

.filtros {
  display: flex;
  gap: 12px;
}

Usa clase cuando ese estilo podria servirte en mas de un lugar.

3. ID

Cuando quieres apuntar a una sola parte muy puntual

Los IDs llevan id="..." en HTML. En CSS se llaman con # adelante.

HTML

<section id="contacto">...</section>

CSS

#contacto {
  scroll-margin-top: 24px;
}

Normalmente un id se usa una sola vez en toda la pagina.

4. Todo

Cuando quieres tocar absolutamente todo

El asterisco * apunta a todos los elementos de la pagina al mismo tiempo.

HTML

<body>
  <h1>Titulo</h1>
  <p>Texto</p>
</body>

CSS

* {
  margin: 0;
}

Se usa mucho para dar una base general antes de empezar a estilizar.

5. Dentro de otro

Cuando quieres tocar elementos que estan adentro

Si escribes un selector y luego otro, CSS entiende que hablas del elemento que esta dentro del primero.

HTML

<div>
  <p>Este cambia</p>
</div>

CSS

div p {
  color: blue;
}

Esto significa: "toca los p que esten dentro de un div".

6. Hijo directo

Cuando quieres tocar solo el que cuelga directo

El simbolo > sirve para marcar que quieres solo al hijo directo, no a cualquier nieto o mas abajo.

HTML

<div>
  <p>Este cambia</p>
</div>

CSS

div > p {
  color: green;
}

Sirve cuando quieres ser mas preciso y no tocar todo lo que este mas profundo.

7. Combinado

Cuando juntas varias pistas para apuntar mejor

Tambien puedes mezclar etiqueta, clase e id en una misma linea para hablarle a una parte muy especifica.

HTML

<div class="contenedor">
  <h2 id="titulo">Hola</h2>
</div>

CSS

div.contenedor #titulo {
  color: purple;
}

Aca CSS entiende: "busca un div con clase contenedor y adentro toca el #titulo".

Si quieres recordarlo facil, piensa asi: nombre solo es body, punto es .class y numeral es #id.

Despues de ubicar elementos, estas fichas siguen con CSS puro para que tengas mas recursos de escritura, organizacion y responsive sin salir del mismo estilo visual.

8. Pseudoclases

Estados, posiciones y condiciones en CSS

Las pseudoclases empiezan con : y te dejan reaccionar a hover, focus, checked o a posiciones como :nth-child().

Uso rapido

.boton:hover {
  transform: translateY(-2px);
}

CSS completo

.lista li:nth-child(odd) {
  background-color: #eef6f1;
}

.campo:focus-visible {
  outline: 3px solid rgba(31, 92, 74, 0.22);
}

Sirven para estados del mouse, teclado, formularios y recorridos de listas sin agregar clases extra.

9. Pseudoelementos

Partes visuales creadas desde CSS

Los pseudoelementos empiezan con :: y dejan crear adornos, marcadores, placeholders o fragmentos visuales sin mas HTML.

Uso rapido

.etiqueta::before {
  content: "# ";
}

CSS completo

.tarjeta::after {
  content: "";
  position: absolute;
  inset: auto 16px 16px auto;
  width: 12px;
  height: 12px;
  border-radius: 50%;
  background-color: #1f5c4a;
}

.campo::placeholder {
  color: #7b8c9b;
}

Muy utiles para detalles visuales, iconos simples y textos de apoyo como ::placeholder o ::selection.

10. Atributos

Selector por tipo, estado o dato

Los selectores de atributo apuntan a valores concretos como [type="email"], [disabled] o [data-estado="ok"].

Uso rapido

input[type="search"] {
  border-radius: 999px;
}

CSS completo

button[disabled] {
  opacity: 0.55;
  cursor: not-allowed;
}

[data-estado="ok"] {
  border-color: #1f5c4a;
}

Son practicos cuando quieres estilos muy puntuales sin inventar una clase nueva para cada caso.

11. Variables CSS

Colores, espacios y medidas reutilizables

Las variables CSS se definen con --nombre y se usan con var(). Sirven para temas, escalas y cambios rapidos en toda la hoja.

Uso rapido

:root {
  --color-principal: #1f5c4a;
}

CSS completo

:root {
  --color-principal: #1f5c4a;
  --espacio-card: 20px;
}

.boton {
  padding: var(--espacio-card, 16px);
  background-color: var(--color-principal, #173a31);
  color: #f6fffb;
}

Si cambias una variable en :root, el ajuste puede propagarse a toda la interfaz sin reescribir veinte reglas.

12. Media Queries

Responsive desde CSS segun ancho o contexto

Las media queries agrupan reglas que solo se activan cuando se cumple una condicion, por ejemplo un ancho maximo de pantalla.

Uso rapido

@media (max-width: 720px) {
  .grilla {
    grid-template-columns: 1fr;
  }
}

CSS completo

.hero {
  padding: 32px;
}

.propiedades-grid {
  grid-template-columns: repeat(3, minmax(0, 1fr));
}

@media (max-width: 720px) {
  .hero,
  .bloque {
    padding: 20px;
  }

  .propiedades-grid {
    grid-template-columns: 1fr;
  }
}

Te sirven para acomodar grillas, espacios, tipografias y layout cuando el ancho disponible cambia.

13. Unidades

Medidas fluidas y formulas practicas

CSS acepta unidades relativas como rem, %, vw y funciones como clamp() o calc().

Uso rapido

h2 {
  font-size: clamp(1.4rem, 2vw, 2.2rem);
}

CSS completo

.pagina {
  width: calc(100% - 32px);
  max-width: min(1180px, 100%);
}

.hero h1 {
  font-size: clamp(2rem, 5vw, 3.4rem);
}

Mezclar unidades relativas con funciones te ayuda a lograr una UI mas flexible sin llenar todo de numeros fijos.

14. Shorthand

Atajos para escribir varias cosas en una sola linea

Muchas propiedades tienen una version corta que junta varios valores en una sola regla, como border, margin o background.

Uso rapido

.card {
  border: 1px solid #d8d1c4;
}

CSS completo

.card {
  margin: 20px auto;
  padding: 16px 20px;
  border: 1px solid #d8d1c4;
  background: linear-gradient(180deg, #fffefb 0%, #f5faf7 100%);
}

Van bien para escribir mas rapido, pero conviene saber que cada valor ocupa una parte distinta del shorthand.

15. Movimiento

Transiciones y animaciones con CSS

transition suaviza cambios entre estados y animation repite un movimiento usando @keyframes.

Uso rapido

.boton {
  transition: transform 0.2s ease;
}

CSS completo

.boton {
  transition:
    transform 0.2s ease,
    box-shadow 0.2s ease;
}

.boton:hover {
  transform: translateY(-2px);
}

.badge {
  animation: demoPulse 1.4s ease-in-out infinite;
}

Para interfaces comunes, primero suele alcanzar con transition; deja animation para efectos mas autonomos o repetidos.
